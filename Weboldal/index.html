ER-modell

Az adatbázis fő egyedtípusai: Páciens, Vizsgálat, Betegség, Gyógyszer, valamint a kapcsolatokat leíró egyedek: Diagnózis, Mit_kap (betegség–gyógyszer), Mire_érzékeny (páciens–gyógyszer).

Egyedek és attribútumok
  • Páciens(Pid, név, cím, telefon)
  • Vizsgálat(Vid, Pid, dátum)
  • Betegség(Bnév)
  • Gyógyszer(Gynév)
  • Mit_kap(Bnév, Gynév)
  • Mire_érzékeny(Pid, Gynév)
  • Diagnózis(Vid, dátum, Bnév)
Kapcsolatok
  • Páciens – Vizsgálat: 1–N (egy páciensnek több vizsgálata lehet)
  • Vizsgálat – Betegség: N–N (egy vizsgálaton több betegség diagnosztizálható)
  • Betegség – Gyógyszer: N–N (egy betegségre több gyógyszer is felírható)
  • Páciens – Gyógyszer: N–N (egy páciens több gyógyszerre is lehet érzékeny)
  • Vizsgálat egyértelmű: (Vid) vagy (dátum + vizsgálat kódja)

Relációs modell

Az ER-modellből származtatott relációk, elsődleges és idegen kulcsokkal.

Tábla Attribútumok Elsődleges kulcs Idegen kulcsok
Paciens Pid, nev, cim, telefon Pid –
Vizsgalat Vid, Pid, datum Vid Pid → Paciens(Pid)
Betegseg Bnev Bnev –
Gyogyszer Gynev Gynev –
Mit_kap Bnev, Gynev (Bnev, Gynev) Bnev → Betegseg(Bnev), Gynev → Gyogyszer(Gynev)
Mire_erzekeny Pid, Gynev (Pid, Gynev) Pid → Paciens(Pid), Gynev → Gyogyszer(Gynev)
Diagnózis Vid, datum, Bnev (Vid, Bnev) Vid → Vizsgalat(Vid), Bnev → Betegseg(Bnev)

Adatbázis létrehozása – SQL (MySQL / phpMyAdmin)

Az alábbi kód bemásolható phpMyAdmin-ba (SQL fül), és futtatva létrehozza a táblákat és a mintaadatokat.

CREATE DATABASE IF NOT EXISTS haziorvosi
  DEFAULT CHARACTER SET utf8mb4
  COLLATE utf8mb4_hungarian_ci;
USE haziorvosi;

CREATE TABLE Paciens (
  Pid   BIGINT PRIMARY KEY,
  nev   VARCHAR(100) NOT NULL,
  cim   VARCHAR(200),
  telefon VARCHAR(50)
);

CREATE TABLE Betegseg (
  Bnev VARCHAR(100) PRIMARY KEY
);

CREATE TABLE Gyogyszer (
  Gynev VARCHAR(100) PRIMARY KEY
);

CREATE TABLE Vizsgalat (
  Vid   INT PRIMARY KEY,
  Pid   BIGINT NOT NULL,
  datum DATE NOT NULL,
  CONSTRAINT fk_vizsgalat_paciens
    FOREIGN KEY (Pid) REFERENCES Paciens(Pid)
);

CREATE TABLE Mit_kap (
  Bnev VARCHAR(100) NOT NULL,
  Gynev VARCHAR(100) NOT NULL,
  PRIMARY KEY (Bnev, Gynev),
  CONSTRAINT fk_mitkap_betegseg
    FOREIGN KEY (Bnev) REFERENCES Betegseg(Bnev),
  CONSTRAINT fk_mitkap_gyogyszer
    FOREIGN KEY (Gynev) REFERENCES Gyogyszer(Gynev)
);

CREATE TABLE Mire_erzekeny (
  Pid BIGINT NOT NULL,
  Gynev VARCHAR(100) NOT NULL,
  PRIMARY KEY (Pid, Gynev),
  CONSTRAINT fk_erz_paciens
    FOREIGN KEY (Pid) REFERENCES Paciens(Pid),
  CONSTRAINT fk_erz_gyogyszer
    FOREIGN KEY (Gynev) REFERENCES Gyogyszer(Gynev)
);

CREATE TABLE Diagnózis (
  Vid   INT NOT NULL,
  datum DATE NOT NULL,
  Bnev  VARCHAR(100) NOT NULL,
  PRIMARY KEY (Vid, Bnev),
  CONSTRAINT fk_diag_vizsgalat
    FOREIGN KEY (Vid) REFERENCES Vizsgalat(Vid),
  CONSTRAINT fk_diag_betegseg
    FOREIGN KEY (Bnev) REFERENCES Betegseg(Bnev)
);

Tesztlekérdezések

Néhány példa lekérdezés az adatbázis működésének ellenőrzésére.

1. Páciensek vizsgálatai és diagnózisai

Melyik páciensnek milyen betegséget diagnosztizáltak egy vizsgálaton?

SELECT p.nev, v.Vid, v.datum, d.Bnev
FROM Paciens p
JOIN Vizsgalat v ON p.Pid = v.Pid
JOIN Diagnózis d ON v.Vid = d.Vid
ORDER BY v.datum, p.nev;
2. Betegségre felírt gyógyszerek

Milyen gyógyszereket kap egy adott betegség?

SELECT m.Bnev AS Betegseg, m.Gynev AS Gyogyszer
FROM Mit_kap m
ORDER BY m.Bnev, m.Gynev;
3. Gyógyszerérzékeny páciensek

Kik érzékenyek valamely gyógyszerre?

SELECT p.nev, me.Gynev
FROM Mire_erzekeny me
JOIN Paciens p ON me.Pid = p.Pid
ORDER BY p.nev;
4. Egy adott napon történt vizsgálatok

1996-02-11-i vizsgálatok listája páciensekkel.

SELECT v.Vid, v.datum, p.nev
FROM Vizsgalat v
JOIN Paciens p ON v.Pid = p.Pid
WHERE v.datum = '1996-02-11'
ORDER BY v.Vid;
